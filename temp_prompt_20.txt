6.1.2 Das Cockpit zerlegen (für Server-Hosting)

Sobald dein Cockpit auf einem Server liegt – sei es bei GitHub Pages, einem klassischen Webhosting-Anbieter oder auf deinem eigenen Raspberry Pi – entfallen die technischen Einschränkungen des Browsers. Über HTTP oder HTTPS kannst du dann problemlos separate CSS-, JavaScript- und HTML-Dateien laden.

Die Vorteile im Überblick:

Wartbarkeit: Änderungen am Design betreffen nur die CSS-Datei, Logik-Änderungen nur die JavaScript-Datei. Du musst nicht mehr in einer riesigen HTML-Datei suchen.

Teamarbeit: Kollegen können an einzelnen Seiten arbeiten, ohne sich gegenseitig den Code zu zerschießen. Jeder kümmert sich um sein Modul.

Ladezeit: Der Browser lädt nur die Dateien, die für die aktuelle Ansicht nötig sind. Das macht dein Cockpit spürbar schneller.

Erweiterbarkeit: Neue Funktionen fügst du als separate Datei hinzu – ohne das Grundgerüst anzufassen. Möchtest du eine Ticket-Seite? Einfach tickets.html erstellen.

Automatisierung: Externe Tools wie n8n oder OpenClaw können CSV-Dateien im gleichen Ordner aktualisieren, ohne die HTML-Struktur zu berühren. Das ist der Schlüssel zur echten Automatisierung.

Die neue Ordnerstruktur (Beispiel):

6.2 Das Cockpit online bereitstellen

Dein Cockpit funktioniert lokal hervorragend. Doch wenn du im Team arbeitest, wollt ihr alle auf den gleichen, aktuellen Datenstand zugreifen. Die Zentrale muss also ins Netz.

Bevor du beginnst, musst du eine wichtige Grundregel des Internets kennen: Möchtest du dein Cockpit als Website aufrufen, musst du deine Hauptdatei zwingend in index.html umbenennen. Webserver sind so programmiert, dass sie exakt diese Datei als Startseite laden. Heißt deine Datei anders (etwa cockpit.html), erkennt der Server sie nicht und dein Bildschirm bleibt leer.

Nun stehen wir vor einem branchenbekannten Problem: Das reine Hochladen einer Webseite ist heute oft kostenlos – aber sobald man ein sicheres „Türschloss“ (einen Passwortschutz) davorlegen möchte, verlangen fast alle Anbieter plötzlich Geld.

Eine extrem wichtige Warnung vorab: Wenn du eine KI fragst, wie du eine HTML-Datei kostenlos mit einem Passwort schützt, rät sie dir oft zu einem kleinen JavaScript-Code, der beim Öffnen ein Passwort-Fenster einblendet. Tu das niemals! Das ist ein reiner „Fake-Schutz“. Jeder, der im Browser auf „Rechtsklick -> Seitenquelltext anzeigen“ klickt, sieht sofort alle deine Finanz- und Prozessdaten sowie das Passwort im Klartext.
